## Описание проекта

Веб‑приложение для **бронирования комнат в отеле**.  
Пользователь может подобрать свободную комнату по датам, забронировать её, управлять своими бронями.  
Приложение поддерживает **две языковые версии (RU/EN)** и отображение цен в разных валютах (RUB / USD / EUR / KZT).

## Основные возможности

- **Работа с комнатами**
  - Просмотр списка комнат (без авторизации).
  - Поля комнаты: номер/название, стоимость за сутки, количество мест, валюта.
  - Фильтрация и сортировка по цене и количеству мест (через API).
- **Поиск и бронирование**
  - Поиск **свободных комнат** в выбранном интервале дат.
  - Оформление бронирования через модальное окно.
  - Отмена брони пользователем или суперюзером.
  - Раздел «Мои брони» для авторизованного пользователя.
- **Авторизация и регистрация**
  - Регистрация нового пользователя.
  - Вход по имени пользователя и паролю.
  - Бронирование доступно **только авторизованным** пользователям.
- **Интерфейс**
  - Переключение языка RU / EN (через Django i18n).
  - Переключение валюты отображения цен: RUB / USD / EUR / KZT (конвертация на фронтенде).

## Стек

- **Backend**: Django 6, Django REST Framework.
- **Аутентификация**: JWT (`djangorestframework-simplejwt`).
- **Фильтрация**: `django-filter`.
- **База данных**:
  - По умолчанию — PostgreSQL (`dj-database-url`, образ `postgres:16` в `docker-compose.yml`).
  - Для быстрого локального запуска используется SQLite (через переменную окружения `USE_SQLITE=1` и скрипт `run.bat`).
- **Frontend**: HTML + CSS + нативный JS (файл `static/js/app.js`).

## Установка и запуск (Windows, PostgreSQL – вариант для проверки)

Этот вариант соответствует требованиям задания (СУБД — PostgreSQL).

1. **Клонируйте/распакуйте** проект в папку, например:

   ```bash
   C:\Users\...\test
   ```

2. Убедитесь, что установлены:
   - **Python 3.10+**
   - **Docker Desktop** (для PostgreSQL).

3. В корне проекта создайте виртуальное окружение (если ещё нет):

   ```bash
   python -m venv venv
   ```

4. Запустите скрипт `run.bat`:

   ```bash
   .\run.bat
   ```

   Скрипт:
   - устанавливает `USE_SQLITE=0` → проект работает на PostgreSQL;
   - активирует `venv`, ставит зависимости из `requirements.txt`;
   - поднимает контейнер `postgres:16` командой `docker compose up -d`;
   - выполняет `python manage.py migrate` и `collectstatic`;
   - запускает сервер: `http://127.0.0.1:8000`.

5. Откройте браузер и перейдите по адресу:
   - Пользовательский интерфейс: `http://127.0.0.1:8000/` или `http://127.0.0.1:8000/en/`.
   - Админ‑панель: `http://127.0.0.1:8000/admin/` или `http://127.0.0.1:8000/en/admin/`.

## Быстрый запуск на SQLite (только для локальной разработки)

Если Docker недоступен или нужно просто быстро потестировать фронтенд, можно использовать SQLite.

1. В корне проекта:

   ```bash
   python -m venv venv
   ```

2. Запустите скрипт `run_sqlite.bat`:

   ```bash
   .\run_sqlite.bat
   ```

   Скрипт:
   - устанавливает `USE_SQLITE=1` → проект работает на базе `db.sqlite3`;
   - активирует `venv`, ставит зависимости;
   - выполняет миграции и `collectstatic`;
   - запускает сервер: `http://127.0.0.1:8000`.

## Структура проекта (кратко)

- `config/` – настройки Django, URL‑маршруты, корневой `settings.py`.
- `rooms/` – приложение управляет комнатами:
  - `models.py` – модель `Room` (номер/название, цена, валюта, количество мест).
  - `views.py` – API для просмотра комнат и поиска доступных.
  - `services.py` – логика поиска свободных комнат по датам.
  - `urls.py` – маршруты `/api/rooms/` и `/api/rooms/available/`.
- `bookings/` – приложение для бронирований:
  - `models.py` – модель `Booking` (комната, пользователь, даты, статус и т.п.).
  - `views.py` – API для создания, просмотра и отмены бронирований.
  - `urls.py` – маршруты `/api/bookings/`.
- `templates/index.html` – главная страница с формой поиска, авторизацией и разделом «Мои брони».
- `static/css/style.css` – стили интерфейса.
- `static/js/app.js` – фронтенд‑логика (поиск, бронирование, авторизация, смена языка и валюты).
- `run.bat` – запуск проекта на Windows с PostgreSQL.
- `run_sqlite.bat` – запуск проекта на Windows с SQLite (для локальной разработки).
- `requirements.txt` – список зависимостей Python.

## API (очень кратко)

- **Аутентификация**
  - `POST /api/auth/register/` – регистрация пользователя.
  - `POST /api/auth/login/` – получение JWT‑токена (`access`).
- **Комнаты**
  - `GET /api/rooms/` – список комнат (поддерживает фильтрацию и сортировку).
  - `GET /api/rooms/available/?check_in=YYYY-MM-DD&check_out=YYYY-MM-DD` – свободные комнаты.
- **Бронирования (требуется JWT в `Authorization: Bearer <token>`)**
  - `GET /api/bookings/` – список броней текущего пользователя.
  - `POST /api/bookings/` – создать бронь.
  - `POST /api/bookings/{id}/cancel/` – отменить бронь.

## Полезные советы

- Если что‑то не работает с базой PostgreSQL, сначала проверьте, что контейнер `db` запущен и порт `5432` не занят другим процессом.
- Для отладки удобно запускать через `run.bat` (SQLite), а для финальной проверки требований — через Docker + PostgreSQL.
- Не удаляйте файлы миграций в `rooms/migrations/` и `bookings/migrations/` – Django использует их, чтобы понимать структуру базы.
